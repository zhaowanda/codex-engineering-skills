# FAQ

## Can this repository include private project skills?

No. Keep private overlays, project registries, code indexes, baseline docs, generated cases, and business semantics outside the open-core repository.

## Do all changes need the full delivery workflow?

No. Use the workflow guide and risk governors to choose the required gates. Small documentation changes can use a lightweight path, while code, configuration, security, or release changes need stronger evidence.

## Can these skills replace CI or human review?

No. They structure AI-assisted engineering work and evidence collection, but project CI, maintainers, code owners, and release approvers remain authoritative.

## How should I start?

Run the synthetic example from `docs/getting-started.md`, then try the prompt pack for one-line requests, long PRDs, bugfixes, code review, and release readiness.

# Deprecation Policy

## Notice

Public skills, schemas, CLI commands, prompts, and docs that are planned for removal must be documented here before removal.

## Migration

Provide the replacement name or path, the reason for the change, and the minimum migration steps.

## Compatibility Window

Keep the old contract available for a reasonable compatibility window unless the change is purely internal and not publicly referenced.

## Removal

Remove a deprecated contract only after the migration path is published, tests are updated, and the changelog records the change.
